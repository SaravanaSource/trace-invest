## Architecture inconsistencies / notes

- The repository contains two ingestion flavors: `src/trace_invest/ingestion/*` (yfinance-based fetchers) and `src/trace_invest/data_ingestion/*` (local deterministic loaders). Recommend consolidating into a single ingestion layer with a pluggable backend (local vs external).
- API `backend/app/services/phase2.py` computes signals on demand by reading snapshots and local financial files. This is correct for read-only, auditable behavior but can be expensive for large universes; consider precomputing signals and persisting them to `data/signals/` for fast API responses.
- Frontend pages attempted server-side fetches during build; if backend not running or artifacts missing, pages may 404 during static generation. The orchestration script ensures artifacts are created before starting servers.

## Recommendations

- Consolidate ingestion modules into `src/trace_invest/ingestion/` with deterministic local mode and optional external connectors.
- Persist computed signals and opportunities to disk for fast API responses and reproducibility.
- Add CI job to run `tools/run_phase2_pipeline.py` on schedule and validate with `tools/verify_phase2.py`.
- Expand signal library and add unit tests for signal logic.
